# 安全说明

## 安全风险提示

FinalShell Clone 在安全性方面存在以下已知风险，使用前请仔细评估：

### 密码存储安全

**风险等级：高**

- **DES 算法**：使用过时的 DES 算法（56位密钥），安全强度不足
- **ECB 模式**：使用 ECB 加密模式，相同明文产生相同密文，存在模式攻击风险
- **硬编码密钥**：使用固定密钥 "finalshell" 进行加密，缺乏密钥多样性

**建议**：
- 仅在可信环境中使用
- 定期更换连接密码
- 优先使用 SSH 密钥认证而非密码认证

### 网络通信安全

**风险等级：高**

- **证书验证跳过**：`AllTrustManager` 信任所有 SSL 证书
- **主机名验证跳过**：`HostnameVerifier` 总是返回 true
- **中间人攻击风险**：可能受到网络中间人攻击

**建议**：
- 仅在内网环境使用
- 使用 VPN 或其他安全隧道
- 避免在不可信网络环境中传输敏感数据

### 文件权限安全

**风险等级：中**

- 配置文件可能包含敏感信息
- 日志文件可能记录连接信息
- 临时文件清理机制需要加强

**建议**：
- 设置适当的文件系统权限
- 定期清理临时文件和日志
- 加密敏感配置文件

## 安全最佳实践

### 1. 环境隔离
- 在隔离的网络环境中使用
- 避免在生产环境中存储敏感连接信息

### 2. 访问控制
- 限制应用程序的文件系统访问权限
- 使用专门的用户账户运行应用

### 3. 数据保护
- 定期备份配置数据
- 使用外部密钥管理系统
- 启用操作系统级别的磁盘加密

### 4. 网络安全
- 通过 VPN 连接远程服务器
- 使用防火墙限制网络访问
- 监控异常网络活动

### 5. 定期更新
- 关注安全公告和更新
- 及时修复已知安全漏洞
- 更新依赖库版本

## 合规建议

本软件**不适用**于以下场景：
- 金融行业生产环境
- 处理个人隐私数据
- 政府机构关键系统
- 需要 SOX/PCI-DSS 合规的环境
